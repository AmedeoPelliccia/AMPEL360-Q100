## KNOT/KNU Methodology

The Q100 program uses KNOT (Knowledge Nexus of Tangible-uncertainties) and 
                    KNU (Knowledge Nugget Units) to manage knowledge creation and uncertainty reduction.


### What is a KNOT?

A KNOT is a cluster of related uncertainties requiring coordinated resolution. 
                        Each KNOT has:

Unique ID (e.g., KNOT-00-00-001)
                            Residual uncertainty score (0-100, target â‰¤15)
                            Set of KNUs for resolution
                            TT token pool for incentivization

Example KNOT: KNOT-00-00-001 
                        (Controlled Terminology Foundation) includes 12 KNUs covering glossary 
                        requirements, database schema, term type taxonomy, validation tests, and

## Document Numbering Conventions

All Q100 documents follow strict naming conventions:


### KNU Identifier Format

KNU-{ATA}-{SUBSYS}-{SEQ}-{TYPE}-{NUM}

Where:
  ATA     = 2-digit ATA chapter (00-99)
  SUBSYS  = 2-digit subsystem (00-99)
  SEQ     = 3-digit sequence (001-999)
  TYPE    = Artifact type (REQ, ICD, ANA, etc.)
  NUM     = 3-digit artifact number (001-999)

Examples:
  KNU-00-00-001-REQ-001  = General, subsys 00, seq 001, Requirements #001
  KNU-71-00-003-ICD-002  = Power Plant, general, seq 003, ICD #002
  KNU-28-10-001-TEST-001 = Fuel, LHâ‚‚ storage, seq 001, Test #001


### S1000D DMC Format

S1000D Data Module Codes (DMCs) follow S1000D standard:

DMC-{MODEL}-{SYSDIFF}-{SYS}-{SUBSYS}-{SUBSUBSYS}{ASSY}-{DISASSY}{VAR}-{INFO}{VAR}-{LOC}_{LEARN}-{EVENT}_{LANG}.{EXT}

Example:
DMC-AMPEL360-00-00-00-00A-042A-A_001-00_EN-US.XML

Where:
  AMPEL360 = Model identifier
  00-00-00-00A = System breakdown
  042A-A = Info code + variant (042 = Usage Guide)
  001-00 = Learn code + event
  EN-US = Language (English-US)
  XML = File format


## SSOT vs PUB Distinction

Understanding the difference between SSOT and PUB is critical:



| Aspect | SSOT (Single Source of Truth) | PUB (Publications) |
| --- | --- | --- |
| Purpose | Engineering artifacts, authoritative source | Derived documentation for end users |
| Format | Markdown, YAML, CSV, JSON | S1000D XML, PDF, HTML (IETM) |
| Lifecycle | LC01-LC13 categories | AMM, CMM, IPC, TRN, etc. |
| Edit Control | Direct editing by engineers | Generated from SSOT, manual edits discouraged |
| Example | KNU-28-00-001-REQ-001_fuel-system-requirements.md | DMC-AMPEL360-28-00-00-00A-001A-A_001-00_EN-US.XML |


Key Rule: SSOT is the authoritative 
                    source. PUB documents are derived via transformation tools or CIR export. 
                    Never manually edit both SSOT and PUB for the same content.
